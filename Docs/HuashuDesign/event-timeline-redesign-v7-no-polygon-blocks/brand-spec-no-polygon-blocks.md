# 事件时间轴 App · No Polygon Blocks v7 视觉规范
> 日期：2026-04-28  
> 范围：视觉设计交付，不修改 Flutter 代码  
> 关键要求：取消不规则多边形色块；不使用渐变色；不使用黑色或深色填充块；页面逻辑、入口、文案和业务规则保持当前截图语义

## 设计方向

方向名：**Light Card Timeline / 轻卡片时间轴**

- 基础是纯色浅蓝白界面，保持极简、轻松和可读。
- 不再使用装饰性多边形、斜切色块或有机色块。
- 重点状态用更克制的视觉层级表达：浅色底、细边框、红点、状态标签、图标和文字权重。
- 信息结构和当前截图保持一致，不新增入口或业务动作。

## 重点状态表达规则

- 新动态：底部“我的关注”红点 + 对应卡片红点 + 最新动态摘要高亮。
- 当前专题：轻蓝底、蓝色细边框、当前状态标签，不使用大面积深色。
- 置顶专题：使用置顶图标和轻标签，不增加独立管理入口。
- 准备中：薄荷浅底和状态标签表达进度，不做复杂 loading 页面。
- 空状态、失败提示、额度提示：使用浅色卡片、线性图标、短文案和明确主按钮。
- 普通列表项保持轻量，避免装饰噪音。

## 禁用规则

- 不使用 `linear-gradient`、`radial-gradient` 或 `conic-gradient`。
- 不使用 `clip-path` 绘制装饰色块。
- 不使用黑色或深色填充块作为按钮、Toast、导航选中态或重点背景。
- 不新增通知中心、我的专题、编辑、删除或归档入口。

## 文案和逻辑锁定

- 推荐分类：`可能关心 / 当前热门 / 随机看看 / 历史记录`。
- 底部 Tab：`我的关注 / 时间轴 / 推荐`。
- 时间轴更多菜单：`新建时间线 / 分享 / 取消关注 / 时间顺序`。
- 新建流程：`新建时间线 -> 关键词 -> AI 扩写 -> 专题定义确认`。
- 不新增通知中心、我的专题、编辑、删除或归档入口。
