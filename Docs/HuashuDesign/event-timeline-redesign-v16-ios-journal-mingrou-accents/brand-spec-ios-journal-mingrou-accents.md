# 事件时间轴 App · iOS Journal Mingrou Accents v16 视觉规范
> 日期：2026-04-28  
> 范围：视觉设计交付，不修改 Flutter 代码  
> 关键要求：每个页面保留独立主题色，并按参考图配色原理加入跨色相点缀；不使用渐变色；不使用不规则多边形色块；不使用黑色或深色填充块；页面逻辑、入口、文案和业务规则保持当前截图语义

## 配色策略

参考图不是单色主题，而是高明度色相互相穿插：蓝、绿、珊瑚、淡紫、暖黄在奶油白底上形成节奏。v16 的策略是：

- 页面主色负责整体气质和当前路径识别。
- 其他色相只以小面积点缀出现，避免影响阅读。
- 点缀色优先落在非正文区域：排名章、小图标、统计点、节点数量点、chip、状态标签。
- 新动态和错误仍以珊瑚/红点为准，避免状态语义混乱。

## 页面主题与点缀

- **我的关注**：珊瑚桃色主色  
  点缀天蓝、青柠、淡紫，用于普通专题图标、日期、小 chip。
- **时间轴**：青柠绿主色  
  点缀天蓝、珊瑚、暖黄，用于不同节点数量点和统计小点。
- **推荐**：天蓝湖蓝主色  
  点缀珊瑚、青柠、淡紫，用于不同推荐卡的排名章和轻状态。

## 组件原则

- 推荐卡片：主按钮和分段控件跟随推荐蓝；不同排名章穿插珊瑚、青柠、淡紫。
- 我的关注卡片：页面主色为珊瑚；普通专题图标可使用蓝/绿/紫点缀；新动态仍保持珊瑚提醒。
- 时间轴页：时间线和主统计为青柠；不同节点数量点用蓝、珊瑚、暖黄区分节奏。
- Chip / 标签：核心状态跟随页面主题，辅助 chip 允许用淡紫、青柠、暖黄交替。
- 所有点缀面积要小于正文面积，不能抢标题和摘要。

## 禁用规则

- 不使用 `linear-gradient`、`radial-gradient` 或 `conic-gradient`。
- 不使用 `clip-path` 绘制装饰色块。
- 不使用黑色或深色填充块作为按钮、Toast、导航选中态或重点背景。
- 不新增通知中心、我的专题、编辑、删除或归档入口。

## 文案和逻辑锁定

- 推荐分类：`可能关心 / 当前热门 / 随机看看 / 历史记录`。
- 底部 Tab：`我的关注 / 时间轴 / 推荐`。
- 时间轴更多菜单：`新建时间线 / 分享 / 取消关注 / 时间顺序`。
- 新建流程：`新建时间线 -> 关键词 -> AI 扩写 -> 专题定义确认`。
