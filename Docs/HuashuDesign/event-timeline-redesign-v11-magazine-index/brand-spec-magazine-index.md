# 事件时间轴 App · Magazine Index v11 视觉规范
> 日期：2026-04-28  
> 范围：视觉设计交付，不修改 Flutter 代码  
> 关键要求：清爽杂志索引风；新闻文字阅读优先；图标低占比；不使用渐变色；不使用不规则多边形色块；不使用黑色或深色填充块；页面逻辑、入口、文案和业务规则保持当前截图语义

## 设计方向

方向名：**Magazine Index Timeline / 杂志索引时间轴**

- 把 App 看成一份轻量专题索引，而不是新闻客户端首页。
- 推荐和关注列表以文字层级为主：标题、摘要、日期、最新动态优先。
- 图标只保留功能识别，不做大面积视觉占位。
- 使用细分隔线、小编号、浅色状态标签和留白组织阅读节奏。
- 时间轴保留日期列，但节点卡片更像目录条目，便于扫读。

## 组件原则

- 推荐卡片：采用索引行结构，左侧小编号，右侧轻按钮，正文列占主宽度。
- 我的关注卡片：普通项用细分隔；新动态和当前专题才使用浅色底和细边框。
- 时间轴节点：保留左侧日期和数量点，右侧节点内容用目录条目式排版。
- 推荐分类：四等分轻分段控件，文案固定为 `可能关心 / 当前热门 / 随机看看 / 历史记录`。
- 新动态：保留底部红点 + 卡片红点 + 最新动态浅蓝高亮。
- 顶部和底部导航：图标保持小体量，不抢正文注意力。

## 禁用规则

- 不使用 `linear-gradient`、`radial-gradient` 或 `conic-gradient`。
- 不使用 `clip-path` 绘制装饰色块。
- 不使用黑色或深色填充块作为按钮、Toast、导航选中态或重点背景。
- 不新增通知中心、我的专题、编辑、删除或归档入口。

## 文案和逻辑锁定

- 推荐分类：`可能关心 / 当前热门 / 随机看看 / 历史记录`。
- 底部 Tab：`我的关注 / 时间轴 / 推荐`。
- 时间轴更多菜单：`新建时间线 / 分享 / 取消关注 / 时间顺序`。
- 新建流程：`新建时间线 -> 关键词 -> AI 扩写 -> 专题定义确认`。
