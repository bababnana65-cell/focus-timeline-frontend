# 事件时间轴 App · iOS Journal Mingrou Palette v15 视觉规范
> 日期：2026-04-28  
> 范围：视觉设计交付，不修改 Flutter 代码  
> 关键要求：按参考图配色修改 iOS Journal 版；三大主页面各自主题色；标签颜色跟随主题；不使用渐变色；不使用不规则多边形色块；不使用黑色或深色填充块；页面逻辑、入口、文案和业务规则保持当前截图语义

## 参考图配色分析

- **底色**：不是纯白，而是偏暖的奶油白，带一点纸面温度。
- **主色关系**：高明度天蓝、青柠绿、珊瑚红橙并置，颜色强但使用面积被切碎，不压迫。
- **叠色感**：图中通过半透明块面叠加形成丰富中间色；App 中不直接做大装饰块，而是用浅底、标签和状态色模拟这种叠色层次。
- **辅助色**：淡紫、暖黄、棕橄榄作为小面积补充，适合 chip、额度提示、辅助状态。
- **适配原则**：阅读型 App 不能照搬大面积彩色构成，只抽取配色关系和节奏。

## 页面主题色

- **我的关注**：珊瑚桃色  
  用于未读红点、新动态、关注列表状态和轻提醒。
- **时间轴**：青柠绿  
  用于进展节点、时间线、统计卡和准备中状态。
- **推荐**：天蓝湖蓝  
  用于推荐发现、分段控件选中态、关注按钮和查看按钮。

## 组件原则

- 顶部栏：保留入口位置，右上新建按钮跟随当前页面主题。
- 底部 Tab：选中态跟随当前页面主题色；未读红点保持珊瑚。
- 推荐卡片：推荐页整体使用天蓝湖蓝，排名章、关注按钮、查看按钮同主题。
- 我的关注卡片：普通项轻暖底，新动态卡片用珊瑚桃色浅底和红点。
- 时间轴页：统计卡、时间线、节点数量点使用青柠绿体系。
- Chip / 标签：核心状态跟随页面主题，准备中用青柠，错误/额度提示用珊瑚或暖黄。

## 禁用规则

- 不使用 `linear-gradient`、`radial-gradient` 或 `conic-gradient`。
- 不使用 `clip-path` 绘制装饰色块。
- 不使用黑色或深色填充块作为按钮、Toast、导航选中态或重点背景。
- 不新增通知中心、我的专题、编辑、删除或归档入口。

## 文案和逻辑锁定

- 推荐分类：`可能关心 / 当前热门 / 随机看看 / 历史记录`。
- 底部 Tab：`我的关注 / 时间轴 / 推荐`。
- 时间轴更多菜单：`新建时间线 / 分享 / 取消关注 / 时间顺序`。
- 新建流程：`新建时间线 -> 关键词 -> AI 扩写 -> 专题定义确认`。
