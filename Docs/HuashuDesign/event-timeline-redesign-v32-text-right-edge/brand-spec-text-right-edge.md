# 事件时间轴 App · Text Right Edge v32 视觉规范
> 日期：2026-04-29  
> 范围：基于 v31 的主要阅读文本列对齐调整，不修改 Flutter 代码  
> 关键调整：我的关注、时间轴、推荐的主要文本右边界统一

## 本版调整

v32 只调整文本列右侧对齐：

- 推荐页卡片正文右边界作为基准。
- 我的关注卡片标题、关注方向、最新节点消息右边界向推荐页基准对齐。
- 时间轴节点标题和节点摘要右边界向推荐页基准对齐。
- 时间轴节点箭头保留，但向外侧让出阅读列。
- 继承 v31：当前专题不显示“正在查看”文字标签。

## 组件规则

- `rec` 正文列：作为 390px 手机画板的基准右边界。
- `topic`：左右内边距统一为 16px，正文列右边界与推荐卡片一致。
- `timeline .node h3 / p`：向右收敛到推荐页同一条右边界。
- `timeline .node .chev`：保留交互暗示，右移避免压缩正文列。

## 继承规则

- 推荐卡片继承 v30：左侧原图标保留，已关注放到图标下方。
- 当前专题继承 v31：当前状态只通过卡片态表达。
- 时间轴顶部统计区保持单行胶囊。
- 时间轴节点圆点保持 v23 放大样式。
- 推荐分类：`可能关心 / 当前热门 / 随机看看 / 历史记录`。
- 底部 Tab：`我的关注 / 时间轴 / 推荐`。
- 时间轴更多菜单：`新建时间线 / 分享 / 取消关注 / 时间顺序`。
- 不新增通知中心、我的专题、编辑、删除或归档入口。
