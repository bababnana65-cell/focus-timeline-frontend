# 事件时间轴 App · iOS Journal Mingrou Semantic v17 视觉规范
> 日期：2026-04-28  
> 范围：视觉设计交付，不修改 Flutter 代码  
> 关键要求：每个页面保留独立主题色；跨色相点缀按语义统一；同类图标和同类文字颜色一致；不使用渐变色；不使用不规则多边形色块；不使用黑色或深色填充块；页面逻辑、入口、文案和业务规则保持当前截图语义

## 颜色原则

v17 保留参考图的趣味配色，但把 v16 的“按条目变化”收敛为“按语义变化”：

- 同一页面有主色，用于页面路径识别。
- 跨色相点缀只分配给不同组件类型或不同业务语义。
- 同类型图标不能在不同卡片里随机变色。
- 同类型文字必须保持统一颜色。
- 未读/新动态摘要是状态例外，可以使用提醒色。

## 页面主题

- **我的关注**：珊瑚桃色主色  
  置顶图标统一淡紫；普通关注图标统一天蓝；准备中图标统一青柠；新动态摘要统一珊瑚。
- **时间轴**：青柠绿主色  
  时间线、统计主色、普通节点数量点统一青柠；重大节点数量点统一珊瑚；节点摘要文字统一灰蓝。
- **推荐**：天蓝湖蓝主色  
  推荐排名章统一天蓝；关注按钮和查看按钮统一推荐蓝；关注人数图标统一青柠。

## 组件规则

- 推荐排名章：同一推荐页内颜色统一。
- 关注人数图标：同一页面内颜色统一。
- 置顶专题图标：统一淡紫。
- 普通专题图标：统一天蓝。
- 准备中专题图标：统一青柠。
- 时间轴节点摘要：统一 `--muted`。
- 未读最新动态摘要：统一珊瑚提醒色。
- Chip / 标签：同一语义使用同一色，不按位置换色。

## 禁用规则

- 不使用 `linear-gradient`、`radial-gradient` 或 `conic-gradient`。
- 不使用 `clip-path` 绘制装饰色块。
- 不使用黑色或深色填充块作为按钮、Toast、导航选中态或重点背景。
- 不新增通知中心、我的专题、编辑、删除或归档入口。

## 文案和逻辑锁定

- 推荐分类：`可能关心 / 当前热门 / 随机看看 / 历史记录`。
- 底部 Tab：`我的关注 / 时间轴 / 推荐`。
- 时间轴更多菜单：`新建时间线 / 分享 / 取消关注 / 时间顺序`。
- 新建流程：`新建时间线 -> 关键词 -> AI 扩写 -> 专题定义确认`。
