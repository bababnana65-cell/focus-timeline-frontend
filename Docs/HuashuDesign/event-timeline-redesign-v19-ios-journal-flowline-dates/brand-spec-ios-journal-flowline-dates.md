# 事件时间轴 App · iOS Journal Flowline Dates v19 视觉规范
> 日期：2026-04-28  
> 范围：基于 v17 的视觉设计调整，不修改 Flutter 代码  
> 关键调整：时间轴改为“日期列 + 节点流线 + 事件卡片”；保留 v17 的页面主题色、语义色规则、文案、入口和业务规则

## 本版调整

v19 不重做整套视觉，只在 v17 基础上修正时间轴阅读结构：

- 左侧显示每个节点日期，日期在事件卡片前方。
- 中间显示连续节点流线，不再只是空白竖线。
- 节点数量圆点挂在流线上，形成“事件流”的节奏。
- 右侧仍是原事件节点卡片，保留标题、摘要和进入箭头。
- 重大节点圆点使用珊瑚提醒色，普通节点圆点使用时间轴主题色。

## 时间轴组件规则

- 日期文字：统一天蓝色，右对齐，保持小字号但清晰可读。
- 流线：统一青柠绿，贯穿节点组，不使用渐变。
- 流线底色：极浅青柠色圆角轨道，帮助用户识别“节点流”区域。
- 节点摘要：统一 `--muted`，不因节点重要程度变色。
- 重大节点数量点：统一珊瑚色。
- 普通节点数量点：统一时间轴主题青柠色。
- 卡片标题：保持正文主色，优先阅读。

## 继承 v17 的页面主题

- **我的关注**：珊瑚桃色主色  
  置顶图标统一淡紫；普通关注图标统一天蓝；准备中图标统一青柠；新动态摘要统一珊瑚。
- **时间轴**：青柠绿主色  
  时间线、统计主色、普通节点数量点统一青柠；重大节点数量点统一珊瑚；节点摘要文字统一灰蓝。
- **推荐**：天蓝湖蓝主色  
  推荐排名章统一天蓝；关注按钮和查看按钮统一推荐蓝；关注人数图标统一青柠。

## 禁用规则

- 不使用 `linear-gradient`、`radial-gradient` 或 `conic-gradient`。
- 不使用 `clip-path` 绘制装饰色块。
- 不使用黑色或深色填充块作为按钮、Toast、导航选中态或重点背景。
- 不新增通知中心、我的专题、编辑、删除或归档入口。

## 文案和逻辑锁定

- 推荐分类：`可能关心 / 当前热门 / 随机看看 / 历史记录`。
- 底部 Tab：`我的关注 / 时间轴 / 推荐`。
- 时间轴更多菜单：`新建时间线 / 分享 / 取消关注 / 时间顺序`。
- 新建流程：`新建时间线 -> 关键词 -> AI 扩写 -> 专题定义确认`。
