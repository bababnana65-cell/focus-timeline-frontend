# 事件时间轴 App · Ticket Cards v9 视觉规范
> 日期：2026-04-28  
> 范围：视觉设计交付，不修改 Flutter 代码  
> 关键要求：卡片票据风；不使用渐变色；不使用不规则多边形色块；不使用黑色或深色填充块；页面逻辑、入口、文案和业务规则保持当前截图语义

## 设计方向

方向名：**Ticket Cards Timeline / 票据卡片时间轴**

- 把专题和时间线节点设计成“事件记录票据”，强化持续追踪、日期凭证、节点归档的语义。
- 推荐卡片、关注卡片和时间轴节点都有右侧票根区，右侧承载关注/查看/进入等操作。
- 票据感来自细虚线、圆形切口、日期章和轻状态章，不靠重装饰。
- 视觉仍保持年轻、轻松、移动端优先，不走传统新闻客户端或后台系统风。
- 不新增任何当前产品未定义的入口或业务动作。

## 组件原则

- 顶部栏：保留原入口位置，按钮用浅色描边方圆按钮。
- 底部 Tab：白色轻浮层，选中态为浅蓝胶囊，不用深色块。
- 推荐分类：四等分轻分段控件，文案固定为 `可能关心 / 当前热门 / 随机看看 / 历史记录`。
- 推荐卡片：左侧为排名/推荐章，右侧为虚线票根，承载关注状态和查看按钮。
- 我的关注卡片：左侧为专题章，右侧为进入票根；新动态用红点和高亮摘要。
- 时间线节点：左侧日期保留，节点卡片改为票据凭证形态，右侧箭头区用虚线分隔。
- Sheet / 弹层：保持轻表单，不做复杂后台表单。

## 禁用规则

- 不使用 `linear-gradient`、`radial-gradient` 或 `conic-gradient`。
- 不使用 `clip-path` 绘制装饰色块。
- 不使用黑色或深色填充块作为按钮、Toast、导航选中态或重点背景。
- 不新增通知中心、我的专题、编辑、删除或归档入口。

## 文案和逻辑锁定

- 推荐分类：`可能关心 / 当前热门 / 随机看看 / 历史记录`。
- 底部 Tab：`我的关注 / 时间轴 / 推荐`。
- 时间轴更多菜单：`新建时间线 / 分享 / 取消关注 / 时间顺序`。
- 新建流程：`新建时间线 -> 关键词 -> AI 扩写 -> 专题定义确认`。
