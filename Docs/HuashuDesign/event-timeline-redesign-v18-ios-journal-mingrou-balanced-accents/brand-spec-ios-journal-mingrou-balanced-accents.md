# 事件时间轴 App · iOS Journal Mingrou Balanced Accents v18 视觉规范
> 日期：2026-04-28  
> 范围：视觉设计交付，不修改 Flutter 代码  
> 关键要求：每个页面保留独立主题色；参考图式跨色相点缀只按语义分配；同类图标和同类文字颜色一致；不使用渐变色；不使用不规则多边形色块；不使用黑色或深色填充块；页面逻辑、入口、文案和业务规则保持当前截图语义

## 颜色原则

v18 继续使用参考图里的“明亮色相并置”思路，但把它收进移动阅读界面：

- 每个主页面只有一个主色，负责导航选中、主按钮、当前状态。
- 其他色相只作为固定语义点缀，不按卡片序号随机变色。
- 同类型图标不能在不同卡片里换颜色。
- 同类型文字必须保持统一颜色。
- 未读/新动态摘要是状态例外，可以使用提醒色。
- 大背景改为清爽浅灰绿，不使用黄色底色。

## 页面主题与点缀

- **我的关注**：珊瑚桃色主色  
  普通关注图标和日期统一天蓝；置顶图标统一淡紫；准备中统一青柠；新动态红点、标签和未读摘要统一珊瑚。
- **时间轴**：青柠绿主色  
  时间线、统计、普通节点数量点统一青柠；节点日期统一天蓝；重大节点数量点统一珊瑚；更多菜单图标统一暖金；节点摘要统一灰蓝。
- **推荐**：天蓝湖蓝主色  
  推荐排名章、分类选中、关注按钮统一天蓝；关注人数统一青柠；查看按钮统一珊瑚；轻量点缀统一淡紫。

## 组件规则

- 推荐排名章：同一推荐页内颜色统一为推荐主题蓝。
- 推荐关注人数图标和文字：统一青柠绿。
- 推荐“查看”按钮：统一珊瑚色，不按推荐类型变化。
- 置顶专题图标：统一淡紫。
- 普通专题图标：统一天蓝。
- 准备中专题图标：统一青柠。
- 我的关注日期和普通最新动态摘要：统一天蓝。
- 未读最新动态摘要：统一珊瑚提醒色。
- 时间轴节点日期：统一天蓝。
- 时间轴节点摘要：统一 `--muted`。
- 时间轴更多菜单图标：统一暖金。
- Chip / 标签：同一语义使用同一色，不按位置换色。

## 禁用规则

- 不使用 `linear-gradient`、`radial-gradient` 或 `conic-gradient`。
- 不使用 `clip-path` 绘制装饰色块。
- 不使用黑色或深色填充块作为按钮、Toast、导航选中态或重点背景。
- 不新增通知中心、我的专题、编辑、删除或归档入口。

## 文案和逻辑锁定

- 推荐分类：`可能关心 / 当前热门 / 随机看看 / 历史记录`。
- 底部 Tab：`我的关注 / 时间轴 / 推荐`。
- 时间轴更多菜单：`新建时间线 / 分享 / 取消关注 / 时间顺序`。
- 新建流程：`新建时间线 -> 关键词 -> AI 扩写 -> 专题定义确认`。
