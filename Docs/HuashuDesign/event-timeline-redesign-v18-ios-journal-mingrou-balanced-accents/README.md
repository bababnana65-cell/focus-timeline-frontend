# 事件时间轴 App 视觉重设计 · iOS Journal Mingrou Balanced Accents v18

本目录是 `huashu-design` 交付物，只做视觉设计，不修改 Flutter 业务代码。

## 本版方向

- 单独新建 v18 目录，没有覆盖 v17 或其他版本。
- 保留每个页面的独立主题色：
  - 我的关注：珊瑚桃色
  - 时间轴：青柠绿
  - 推荐：天蓝湖蓝
- 参考图里的多色趣味改成“语义点缀”：蓝、绿、珊瑚、淡紫、暖金会同时出现，但只绑定固定含义。
- 同样是置顶图标，颜色统一。
- 同样是普通关注图标，颜色统一。
- 同样是推荐排名章，颜色统一。
- 同样是推荐“查看”按钮，颜色统一。
- 同样是时间轴日期文字，颜色统一。
- 同样是时间轴节点摘要文字，颜色统一。
- 新动态未读摘要允许使用提醒色。
- 不使用渐变色。
- 不使用不规则多边形色块。
- 不使用黑色或深色填充块；深色只用于正文文字和必要线性图标。
- 保留当前截图里的入口、页面结构、按钮文案、分类文案和业务规则。

## 文件

- `event-timeline-ios-journal-mingrou-balanced-accents.html`：高保真移动端设计板。
- `brand-spec-ios-journal-mingrou-balanced-accents.md`：平衡点缀色视觉规范。
- `screenshots/design-board.png`：完整设计板 PNG。
- `screens/*.png`：单页高保真 PNG。
- `figma-handoff.json`：Figma 重建参考 JSON，不是原生 `.fig` 文件。

## 覆盖页面

- 推荐页：当前热门 / 可能关心 / 随机看看 / 历史记录
- 我的关注：普通列表 / 有新动态 / 当前专题 / 空列表
- 时间轴：正常节点列表 / 更多菜单 / 正在准备中 / 初始化失败 / 空时间线
- 新建时间线：输入关键词 / AI 扩写后确认
- 手机号验证弹层
- 关注上限提示
- 刷新失败轻提示
- 组件规范
