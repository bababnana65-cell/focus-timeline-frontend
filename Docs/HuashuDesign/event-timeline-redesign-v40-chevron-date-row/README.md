# 事件时间轴 App 视觉重设计 · Chevron Date Row v40

本目录是 `huashu-design` 交付物，只做视觉设计，不修改 Flutter 业务代码。

## 本版方向

- 单独新建 v40 目录，没有覆盖 v39 或其他版本。
- 以 v39 为基础，只调整时间轴节点右侧 `>` 控件的位置和尺寸。
- 时间轴节点标题和摘要继续保持 `264px`，右边界与“我的关注”专题标题一致。
- 右侧 `>` 箭头按钮调整为 `22px`，放在日期胶囊同一行右侧，不再挤占标题段落。
- 保留 v38 的长文案换行检查。
- 专题标题、关注方向、最新摘要都换成自然换行的长文案，至少能看到两行排版。
- 推荐卡片和我的关注卡片都使用长文案，方便检查真实段落在手机卡片里的行高、间距和右边界。
- 时间轴节点标题和摘要宽度从 `264px` 收到 `242px`，为右侧箭头留出稳定空隙。
- 目标是让时间轴节点文字右间距接近“我的关注”卡片里的“跟踪冲突升级...”和“联调样本...”两行阅读文本。
- 保留 v36 的“我的关注”页右上角新建按钮颜色统一。
- 当前专题、普通列表、有新动态、新建流程背景中的“我的关注”右上角加号，统一为和空列表页一致的珊瑚色按钮。
- 使用 `Docs/HuashuDesign/app icon/timeline_app_icon_mark_transparent.svg` 的透明内标记，替换原 clock 图标。
- 选中态跟随时间轴页绿色主题，未选中态使用柔和灰色，避免深色块。
- 移除底部中间 Tab 的“时间轴”文字，只保留 App 图标。
- 图标放大为独立入口，上下视觉高度对齐“我的关注”的图标 + 文字组合。
- 时间轴节点箭头继续保留 v33 的卡片内原位置。
- 保留 v32 的主要阅读文本右边界对齐规则。
- 保留 v31 的“正在查看”标签移除方案。
- 保留 v30 的推荐页调整：卡片内类型标签隐藏，“已关注”在左侧图标下方。
- 保留未读红点提示。
- 推荐页左滑关注/取消关注逻辑继承 v25。
- 不使用渐变色。
- 不使用不规则多边形色块。
- 不使用黑色或深色填充块。
- 保留当前截图里的入口、页面结构、分类文案和业务规则。
