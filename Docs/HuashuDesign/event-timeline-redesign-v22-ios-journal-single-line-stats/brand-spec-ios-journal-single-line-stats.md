# 事件时间轴 App · iOS Journal Single Line Stats v22 视觉规范
> 日期：2026-04-28  
> 范围：基于 v21 的时间轴局部调整，不修改 Flutter 代码  
> 关键调整：顶部统计区参考推荐页分类切换，改为单行分段胶囊

## 本版调整

v22 只调整时间轴顶部统计区：

- 统计区由三个上下排布小卡片改为一个横向单行胶囊。
- 每项内容为数字和文案同排：`4 时间节点`、`7 动态数量`、`4 重大节点`。
- 控件高度约 44px，与推荐页分类切换的节奏接近。
- 使用轻边框和浅背景，不使用重卡片。
- 保留三个统计项的文案和含义，不改变接口或状态流。

## 统计条组件规则

- 背景：白色 88% 透明感，轻边框。
- 外形：999px 胶囊圆角。
- 分段：3 等分，中间使用极浅分隔线。
- 数字：时间轴主题青柠深色，17px 左右。
- 文案：灰蓝色，11px 左右。
- 单行：不允许数字和文案上下换行。

## 继承规则

- 时间轴节点仍继承 v21：日期标签在卡片顶部，左侧保留细流线和数量圆点。
- 推荐分类：`可能关心 / 当前热门 / 随机看看 / 历史记录`。
- 底部 Tab：`我的关注 / 时间轴 / 推荐`。
- 时间轴更多菜单：`新建时间线 / 分享 / 取消关注 / 时间顺序`。
- 新建流程：`新建时间线 -> 关键词 -> AI 扩写 -> 专题定义确认`。
- 不新增通知中心、我的专题、编辑、删除或归档入口。
