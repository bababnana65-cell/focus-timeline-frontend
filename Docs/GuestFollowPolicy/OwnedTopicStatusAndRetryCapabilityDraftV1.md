# 自建专题状态与重试能力草案 v1

## 一、目标

在当前专题“创建后不可修改、不可删除、不可归档、仅可取消关注”的规则下，
后续新增的管理能力应尽量收敛成最小版本：

- 查看我自己创建的专题
- 查看这些专题当前的初始化状态
- 当初始化失败时，可以手动重试

当前不引入：

- 编辑专题定义
- 删除专题
- 归档专题

## 二、为什么现在做这条线

当前已经具备：

- 登录用户创建专题
- guest create 最小服务端链路
- ownership claim 最小链路
- 阶段 7C 主路径初始化

但用户在创建完专题之后，仍然缺少一个明确的管理视角：

- 我创建了哪些专题
- 它们现在是“正在准备中”还是“可阅读”
- 如果初始化失败了，应该怎么处理

所以这条线的目标不是“更多编辑能力”，而是：

**让用户能看见自己创建的专题状态，并在失败时重试。**

## 三、产品范围

### 1. 我的专题

新增一个“我的专题”视图，返回并展示当前账号拥有的自建专题。

每个专题至少应显示：

- 标题
- 摘要
- 当前状态
- 是否可阅读
- 最近更新时间

### 2. 状态展示

基于当前已有的最小状态语义，前端展示：

- `draft / pending` -> 正在准备中
- `draft / running` -> 初始化中
- `draft / failed` -> 初始化失败
- `active / ready` -> 可正常阅读

### 3. 重试初始化

当专题处于：

- `draft / failed`

允许当前 owner 手动触发一次：

- 重新生成
- 或 重试初始化

这条能力的本质是：

- 不修改专题定义
- 不创建新专题
- 只让当前专题重新进入初始化链路

## 四、明确不做的范围

当前这条能力草案明确不包括：

- 编辑专题定义
- 删除专题
- 归档专题
- 多人协作
- 复杂权限
- 版本历史 UI

这些都不属于当前最小范围。

## 五、后端最小接口建议

### 1. `GET /topics/mine`

用途：

- 返回当前登录账号拥有的自建专题列表

返回内容建议至少包含：

- `topicId`
- `title`
- `summary`
- `status`
- `initializationState`
- `updatedAt`
- `isFollowed`

### 2. `POST /topics/{topicId}/retry-initialization`

用途：

- 仅允许 owner 对失败专题重新触发初始化

适用前提：

- 当前用户是 owner
- 专题处于允许重试的状态

最小返回建议：

- `topicId`
- `status`
- `initializationState`

建议重试后语义：

- `draft / failed` -> `draft / pending`
- 后续再进入 `draft / running`
- 成功后推进到 `active / ready`

## 六、前端最小改动建议

### 1. 增加“我的专题”入口

这不要求重做大结构，只要能让用户进入一个“我创建的专题”视图即可。

### 2. 显示状态

在列表页或专题页上直接显示当前状态：

- 正在准备中
- 初始化中
- 初始化失败
- 可正常阅读

### 3. failed 时显示重试按钮

仅当：

- 当前用户是 owner
- 且状态为 `draft / failed`

才显示：

- 重新生成
- 重试初始化

### 4. 不新增编辑入口

当前前端不要顺手加入：

- 编辑专题
- 删除专题
- 归档专题

## 七、后端权限建议

### `GET /topics/mine`

- 仅登录用户可调用
- 返回当前 owner 维度的数据

### `POST /topics/{topicId}/retry-initialization`

仅满足以下条件时允许：

- 已登录
- 当前用户是 owner
- 专题存在
- 当前状态允许重试

不满足时应返回明确错误。

## 八、与当前已冻结规则的关系

这条能力草案与当前已确认规则完全一致：

- 专题不可修改
- 专题不可删除
- 专题不可归档
- 用户唯一的收口动作仍然是取消关注

也就是说：

“自建专题状态与重试”不是要扩大管理权限，而只是补齐：

- 看状态
- 失败后重试

## 九、建议的实现顺序

1. 后端先做 `GET /topics/mine`
2. 后端再做 `POST /topics/{topicId}/retry-initialization`
3. 前端接“我的专题”最小视图
4. 前端接 failed 状态下的“重试初始化”按钮

## 十、版本规则

当前文档作为 `v1` 草案使用。

如果后续产品决定扩大这条能力，例如：

- 开放编辑定义
- 开放删除
- 开放归档
- 开放更复杂的 owner 管理动作

则统一新增 `v2` 文档，不直接覆盖当前 `v1`。
